There are two sorting algorithms given: insertion sort and merge sort. Given that Python lists have sort built-in, reproducing them precisely here is somewhat silly. So I’ll do some a simple variation.

For example, if we do not require that the insertion sort sorts in place, we can use it to sort iterables into a new list. There is a builtin for that as well, namely sorted. Whatever I do, it seems to be already implemented.

def insertion_sort(it): """insertion_sort(iterable) -> sorted list

Note that this is very much like the builtin function sorted.

>>> insertion_sort((10, 1, 3, 5, 2)) [1, 2, 3, 5, 10]

>>> insertion_sort((31, 41, 59, 26, 41, 58)) [26, 31, 41, 41, 58, 59]

"""

it = iter(it) result = [it.next()] for index, val in enumerate(it): i = index result.append(val) while i >= 0 and result[i] > val: result[i+1], i = result[i], i-1 result[i+1] = val

return result
